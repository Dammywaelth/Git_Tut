Basic Commands

$ git init (initializes a git repository)

$ git branch -m master main (changes your mi=ain branch from master to main)

$ git config --global user.name "your name"

$ git config -1 (list all the details you submitted after git int)

$ git config --global user.email "your email"

$ git add filename (adds a file to the staging area)

$ git add . (adds all files to the staging area)

$ git rm --cached . -r (removes all files from the staging area)

$ git commit -m "your custome message" (saves files in the staging area to git)

$ git status (shows the status of the working tree)

$ git clone (clones an existing repository)

$ git pull (pull the latest changes from the remote repository to your local machine)

$ git push (pushs your local repository to your remote repository)

Other commands

$ git log (gives you a log of your commit history. Each commit history has a hash number)

$ git show hashNumber (shows you what is inside that particular commit)

$ git diff (gives you the difference between the current updated file and the previous commit)

$ git branch (give you the current branch you are working on)

$ git branch -r (gives you the list of branches you have remotely(in github))

$ git branch -a (list all branches both in local & remote end )

$ git branch "yourBranchName" (creates a new branch)

$ git checkout "youBranchName" (to swicth to the specified branch)

$ git checkout - (takes you to the previous branch)

$ git push --set -upstream origin "yourBranchName" (this pushes your new branch to the remote end)

$ git push -u origin "yourBranchName" (This will also push your new branch to the remote end. you can use this or the command above )

to delete a branch, make sure you switch back to the main branch and them run this command: $ git branch -d "youBranchName"

if you are working for a company or collaborating with a team, you will never push your changes to the main branch. You will have to create your own brnach and push your changes into it. then you raise a pull request so that someone will inspect those changes before merging them to the main branch

when merging branches, it is best practice to do it through a pull request

.gitignore file, is a file where other files that you do not want to push to github are kept
